---
title: Простой UART
---

## Обзор

NovumOS использует упрощённую реализацию UART для ввода/вывода в терминал. Это не чип 16550 и не какой-либо другой стандартный UART. Нет регистров состояния, нет FIFO, нет настройки скорости передачи, нет прерываний. Ввод/вывод символов осуществляется опросом (polling), по одному символу за раз.

## Адрес порта

| Порт | Направление | Описание             |
|------|-------------|----------------------|
| 0x00 | OUT         | Отправка символа     |
| 0x00 | IN          | Приём символа        |

## OUT 0x00 — Отправка символа

Запишите символ в порт 0x00, чтобы отправить его в терминал.

```asm
mov al, 'A'
out 0x00, al
```

Символ помещается в буфер передачи эмулятора. Буфер сбрасывается в терминал автоматически. Ждать какого-либо флага состояния не нужно.

## IN AX, 0x00 — Приём символа

Прочитайте символ из порта 0x00. Если символ доступен, он возвращается в `AL`. Если символа нет, возвращается `0`.

```asm
in al, 0x00
cmp al, 0
je no_input
; обработка символа в AL
```

### Цикл опроса

Для ожидания ввода используйте цикл опроса:

```asm
wait_for_input:
    in al, 0x00
    cmp al, 0
    je wait_for_input
; в AL теперь нажатый символ
```

## Поведение эмулятора

- **Буфер TX**: Символы, записанные в порт 0x00, буферизируются эмулятором и отображаются в окне терминала. Буфер сбрасывается после каждой записи.
- **Опрос RX**: Эмулятор перехватывает ввод с клавиатуры. Если пользователь нажал клавишу с момента последнего чтения, клавиша возвращается. В противном случае возвращается `0`.
- **Нет блокировки**: Чтения никогда не блокируют CPU. Если данных нет, чтение возвращает `0` немедленно.
- **Нет прерываний**: Весь ввод должен опрашиваться. IRQ-линия для клавиатуры или последовательного ввода отсутствует.

## Ограничения

- Нет управления потоком (XON/XOFF или RTS/CTS).
- Нет обнаружения break.
- Нет статуса ошибок (фрейминг, переполнение, чётность).
- По одному символу за раз. Нет механизма массовой передачи.
